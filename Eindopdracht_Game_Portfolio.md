# Eindopdracht: Game Portfolio + Builds

## Doel
Maak een **eenvoudige game‑portfolio website** waarin **alle drie je games** worden gepresenteerd  
(**Pong, Pac‑Man en de 2D Platformer**) en waarin je **playable builds** linkt.

- **Windows‑builds via GitHub Artifacts** zijn **verplicht**
- **WebGL‑builds via MA‑Cloud** zijn **verrijkingsopdracht**

Als het verplichte deel werkt, ben je **klaar** -.

---

## Verplicht Deel 1 – Windows Builds maken (Unity)

Je moet een **Windows (Standalone) build** maken van **alle drie** de games:

- Pong  
- Pac‑Man  
- 2D Platformer  

### Eisen
- Elke game is gebuild in Unity  
- Build target is **Windows (Standalone)**  
- Elke build start en is speelbaar  
- Geen errors bij het opstarten  

Gebruik duidelijke mapnamen:
```
pong-windows-build
pacman-windows-build
platformer-windows-build
```

---

## Verplicht Deel 2 – Windows Builds hosten via GitHub Artifacts

Je gebruikt **GitHub Artifacts** om je Windows‑builds te hosten.

### Eisen
- GitHub repository bestaat  
- Elke game heeft een **eigen artifact**  
- Elk artifact:
- Heeft een duidelijke naam (bijv. `Pong – Windows Build`)
- Kan worden gedownload
- Werkt op Windows

>S Let op:
- De game hoeft **niet** in de browser te draaien  
- Downloaden → `.exe` starten → spelen is voldoende  

---

## Verplicht Deel 3 – HTML Portfolio Website

Maak een **eenvoudige HTML‑portfolio pagina**.

### Bestand
```
index.html
```

- Opent in een browser  
- Geen frameworks  
- Geen kapotte HTML  

---

## Verplicht: Inhoud van de Portfolio Pagina

### Algemeen
- Je **naam** bovenaan
- Overzichtelijke structuur
- Pagina laadt zonder fouten

---

### Voor ELKE game (Pong, Pac‑Man, Platformer)

Elke game heeft een eigen sectie met:

1. **Titel**
2. **Korte beschrijving** (2–3 zinnen)
3. **Controls**
4. **Waar je trots op bent**
5. **Link naar de Windows‑build (GitHub Artifact)**
6. **gifs & images zoals in je readme**
7. **link naar je github code**
---

## BEHAALD (verplicht deel)

Je bent **geslaagd** als:

- [ ] Alle drie de games een werkende **Windows‑build** hebben  
- [ ] Alle drie de builds als **GitHub Artifact** bestaan  
- [ ] `index.html` correct opent  
- [ ] Elke game een eigen sectie heeft  
- [ ] Elke sectie linkt naar een **Windows‑build**  
- [ ] Alles staat op **MA‑Cloud**  

- Alles afgevinkt → **KLAAR**

---

# Verrijkingsopdracht – WebGL Build (OPTIONEEL)

Maak van **minstens één game** een **WebGL build** en host deze via **MA‑Cloud**.

### Eisen
- WebGL build start in de browser  
- Game is speelbaar  
- Link toegevoegd aan je portfolio
